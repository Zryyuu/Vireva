
#### 🧩 Fasilitas

```
Lihat fasilitas
```

#### 📜 Kebijakan / Aturan

```
Lihat kebijakan
```

# 3️⃣ MANAJEMEN KAMAR (Paling Penting)

### 3.1 Daftar Kamar

Menampilkan:

* Nama kamar
* Harga
* Status (tersedia / terisi)
* Aksi: Detail | Edit | Hapus

---

### 3.2 Tambah Kamar

```
Klik Tambah
→ Popup Form
→ Isi data kamar
→ Konfirmasi tambah?
    ├─ Ya → Data tersimpan
    └─ Batal → Popup ditutup
```

---

### 3.3 Detail Kamar

Menampilkan:

* Informasi kamar
* Jadwal penggunaan (tanggal + booking)
* Rating kamar (hasil review user)

📌 **Read-only**

---

### 3.4 Edit Kamar

```
Klik Edit
→ Popup Form Edit
→ Konfirmasi update?
    ├─ Ya → Data terupdate
    └─ Batal → Popup ditutup
```

---

### 3.5 Hapus Kamar

```
Klik Hapus
→ Konfirmasi hapus?
    ├─ Ya → Data terhapus
    └─ Batal → Popup ditutup
```

📌 Validasi:

* Kamar **tidak boleh dihapus jika ada booking aktif** (opsional tapi bagus)

---

# 4️⃣ TRANSAKSI

👉 **Admin hanya memantau**, tidak intervensi pembayaran

### 4.1 Daftar Transaksi

Menampilkan:

* Booking ID
* Nama user
* Total bayar
* Status pembayaran
* Metode pembayaran
* Tanggal

```
Admin buka transaksi
→ Sistem ambil data booking + payment
→ Tampilkan daftar
```

---

### 4.2 Refund

```
Admin buka refund
→ Lihat daftar refund
→ Status refund (pending / sukses / gagal)
```

📌 Refund **berdasarkan aturan**, bukan manual approve.

---

# 5️⃣ OPERASIONAL (Ini sudah PAS)

👉 Berisi **aturan & konfigurasi sistem**, bukan data operasional harian

### 5.1 Payment Gateway

```
Admin buka payment gateway
→ Atur API key / mode sandbox / production
→ Simpan konfigurasi
```

📌 Tidak CRUD transaksi ✔️
📌 Hanya konfigurasi ✔️

---

### 5.2 Diskon & Promo

```
Lihat promo
→ Tambah / edit diskon
→ Tentukan:
   - tipe diskon
   - periode
   - syarat
→ Simpan
```

📌 Promo otomatis diterapkan saat booking user.

# 1️⃣ DASHBOARD USER (REVISI ISI)

### Tujuan

👉 **Ringkasan status + shortcut**, bukan halaman utama eksplor

### Konten Dashboard (dipadatkan & relevan)

#### 🔷 Ringkasan Status

* 🛏️ **Booking Aktif**
* ⏳ **Transaksi Berlangsung** (menunggu pembayaran)
* 📅 **Booking Terdekat** (tanggal + kamar)
* ⭐ **Total Riwayat Menginap**

📌 “Booking terdekat” = booking aktif dengan tanggal check-in paling dekat
📌 “Total riwayat” = jumlah booking selesai (opsional, tapi masih masuk akal)

---

#### 🔷 Aksi Cepat

* 🔍 Cari Kamar
* 💳 Lanjutkan Pembayaran (muncul jika ada)
* 🛏️ Booking Lagi

---

### Alur Dashboard

```text
Login
→ Ambil data booking user
→ Hitung status
→ Tampilkan ringkasan + shortcut
```

---

# 2️⃣ MENU: CARI KAMAR (REVISI ALUR)

Ini **menu utama user**, alurnya harus mulus.

## A. Cari Kamar (Normal Flow)

```text
Cari Kamar
→ Tampilkan daftar kamar
→ Lihat detail kamar
→ Klik Booking
→ Isi form booking
→ Konfirmasi
→ Bayar? (opsional sekarang / nanti)
```

---

## B. Cari Cepat (Shortcut)

```text
Cari Cepat
→ Isi tanggal + jumlah tamu
→ Sistem filter kamar
→ Tampilkan hasil
→ Lanjut ke detail kamar
```

📌 Cari cepat **bukan menu terpisah**, tapi fitur di halaman Cari Kamar ✔️

---

## C. Detail Kamar

Menampilkan:

* Foto
* Harga
* Fasilitas
* Rating
* Tombol **Booking**

📌 Tidak ada pembayaran di sini

---

# 3️⃣ MENU: BOOKING SAYA (DIGABUNG TOTAL)

👉 Ini **kontrol pusat user**

## A. Daftar Booking

Menampilkan:

* Semua booking user
* Status:

  * Menunggu pembayaran
  * Aktif
  * Selesai
  * Dibatalkan

Filter:

* Status
* Tanggal

---

## B. Detail Booking

Menampilkan:

* Detail kamar
* Tanggal menginap
* Total harga
* Status pembayaran

### Aksi di Detail Booking

* 💳 Bayar (jika belum)
* ❌ Batalkan (jika masih boleh)
* ⭐ Beri Rating (jika booking selesai)

📌 **Rating hanya muncul di booking selesai**
📌 Tidak perlu menu “Review” terpisah ✔️

---

## Alur Booking & Pembayaran

```text
Booking dibuat
→ Status: menunggu pembayaran
→ User bayar (sekarang / nanti)
→ Payment gateway callback
→ Status booking update otomatis
```
